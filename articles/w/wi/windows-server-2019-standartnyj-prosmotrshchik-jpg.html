Настройка стандартного просмотрщика изображений в Windows Server 2019

  • Настройка ассоциаций файлов через реестр
  • Классический просмотрщик изображений Windows

Администраторы часто сталкиваются с ситуацией, когда после установки операционной системы Windows Server 2019 на рабочем столе или в папках с логами невозможно открыть файлы изображений формата JPG или PNG двойным кликом. Вместоного просмотра появляется системное окно с вопросом «Как вы хотите открыть этот файл?» или запускается браузер, что крайне неудобно при работе с графическими отчетами или скриншотами ошибок.

Проблема кроется в минималистичном подходе Microsoft к серверным редакциям ОС, где по умолчанию отключены мультимедийные компоненты и не заданы ассоциации файлов для пользовательских приложений. Стандартный механизм открытия файлов не может найти подходящее приложение, так как компонент «Фотографии» (Microsoft Photos) не установлен в базовой конфигурации, а классический просмотрщик Windows 7 отсутствует в коде системы.

Для решения этой задачи необходимо вручную установить недостающие компоненты через диспетчер серверов или PowerShell, а затем корректно прописать ассоциации файлов в реестре или через графический интерфейс. Ниже представлен подробный алгоритм действий, позволяющий восстановить функциональность просмотра графики без установки стороннего программного обеспечения.

Установка компонента «Фотографии» через PowerShell

Наиболее эффективным и современным способом восстановления функционала просмотра изображений является установка официального приложения Microsoft Photos из репозиториев Microsoft. В отличие от старых методов, этот подход обеспечивает совместимость с новыми форматами сжатия и поддерживает аппаратное ускорение.

Для начала работы требуется запустить консоль PowerShell с правами администратора. Это можно сделать через меню «Пуск», найдя соответствующий ярлык, или через диспетчер задач, выбрав «Файл» → «Запустить новую задачу» и введя powershell, предварительно поставив галочку «Создать задачу с правами администратора».

В открывшемся окне консоли необходимо выполнить команду для поиска пакета приложений, отвечающего за работу с мультимедиа. Система должна иметь доступ к интернету или к локальному репозиторию WSUS, если политики безопасности запрещают прямое подключение к магазину Microsoft Store.

Get-AppxPackage -AllUsers *Microsoft.Windows.Photos*

Если команда возвращает пустой результат, значит, приложение полностью удалено или никогда не устанавливалось. В этом случае потребуется использовать команду установки из онлайн-каталога, что является стандартной процедурой для серверных сборок.

⚠️

Использование командлетов установки приложений требует активного подключения к серверам Microsoft. Убедитесь, что брандмауэр не блокирует доступ к доменам microsoft.com и windows.com.

Для принудительной установки приложения используется следующий синтаксис, который скачает и зарегистрирует пакет в системе для всех пользователей:

Add-AppxPackage -Register"C:\Program Files\WindowsApps\Microsoft.Windows.Photos_*_x64__8wekyb3d8bbwe\AppxManifest.xml" -DisableDevelopmentMode

Однако, если пакет отсутствует в системе полностью, проще воспользоваться командой поиска и установки через встроенный менеджер пакетов, если он доступен, или скачать Appx-пакет вручную. В большинстве случаев достаточно выполнить команду восстановления:

Get-AppxPackage -AllUsers Microsoft.Windows.Photos | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register"$($_.InstallLocation)\AppXManifest.xml"}

После успешного выполнения скрипта в списке установленных программ появится приложение «Фотографии». Оно автоматически зарегистрирует себя как обработчик основных графических форматов, но в некоторых случаях требуется ручная настройка ассоциаций.

Альтернативный метод через Store для бизнеса

Если сервер находится в корпоративной сети с жесткими ограничениями, установка может производиться через портал Store for Business. Администратор загружает офлайн-инсталлятор приложения Microsoft Photos и развертывает его через групповые политики или SCCM.

Этот метод предпочтителен для сред с закрытым контуром безопасности, где прямой доступ к интернету с сервера запрещен политиками информационной безопасности. Пакет передается локально и устанавливается через PowerShell командой Add-AppxPackage -Path с указанием пути к файлу.

2Введите команду Get-AppxPackage для проверки наличия
3Выполните команду Add-AppxPackage для регистрации
4Проверьте наличие приложения в меню Пуск

Настройка ассоциаций файлов через реестр

В серверных операционных системах часто отсутствует графический интерфейс для выбора программ по умолчанию, либо он работает некорректно без установленных компонентов. В таких случаях наиболее надежным способом является прямое внесение изменений в системный реестр.

Для начала необходимо открыть редактор реестра, выполнив команду regedit в окне «Выполнить» (Win+R). Важно соблюдать осторожность, так как некорректное изменение ключей может привести к нестабильной работе оболочки Explorer.

Необходимо перейти по пути HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\FileExts\.jpg. Здесь создается или модифицируется раздел OpenWithList, где указывается исполняемый файл приложения. Однако более эффективным методом является работа с ключами HKEY_CLASSES_ROOT.

Создайте текстовый файл с расширением.reg и внесите в него следующие данные для принудительной привязки расширений JPG и PNG к стандартному приложению:

Windows Registry Editor Version 5.00


[HKEY_CLASSES_ROOT\.jpg]


@="AppX43hnxtbyyps62jhe9sqpdzxn1790zetc"



[HKEY_CLASSES_ROOT\.png]


@="AppX43hnxtbyyps62jhe9sqpdzxn1790zetc"



[HKEY_CLASSES_ROOT\AppX43hnxtbyyps62jhe9sqpdzxn1790zetc\shell\open\command]


@="\"C:\\Program Files\\WindowsApps\\Microsoft.Windows.Photos_2019.10000.22711.0_x64__8wekyb3d8bbwe\\Microsoft.Windows.Photos.exe\" \"%1\""

Обратите внимание, что версия приложения в пути может отличаться. Для получения актуального пути используйте команду PowerShell (Get-AppxPackage -Name Microsoft.Windows.Photos).InstallLocation.

После применения изменений реестра необходимо перезапустить процесс explorer.exe или выполнить выход из системы и повторный вход, чтобы изменения вступили в силу. Проверка осуществляется двойным кликом по любому файлу изображения.

Параметр реестра Значение Назначение
.jpg AppX... Связь расширения с AppX пакетом
shell\open\command Путь к exe Команда запуска приложения

Использование групповых политик для массового внедрения

В доменной среде настройку просмотрщика можно централизованно распространить на все серверы. Для этого создается объект групповой политики (GPO), который применяет скрипт реестра при загрузке системы или входе пользователя.

Файл реестра размещается в общей сетевой папке с правами только на чтение. В редакторе управления групповыми политиками (gpmc.msc) создается задача в разделе «Конфигурация компьютера» → «Конфигурация Windows» → «Сценарии» → «Загрузка».

📝

При использовании GPO убедитесь, что учетная запись компьютера имеет права на чтение сетевой ресурса с файлом реестра.

Альтернативно, можно использовать предпочтения групповых политик (GPP) в разделе «Реестр», чтобы напрямую прописать необходимые ключи на целевых серверах, что является более гибким методом управления конфигурацией.

Классический просмотрщик изображений Windows

Многие администраторы предпочитают возвращать классический, легкий просмотрщик изображений Windows, который был стандартным в Windows 7. В Windows Server 2019 он формально присутствует в системе, но скрыт из списка доступных программ.

Для активации классического просмотрщика необходимо внести изменения в реестр, добавляя соответствующие ключи для каждого поддерживаемого формата. Это позволяет использовать старый, быстрый интерфейс без загрузки тяжелых библиотек UWP-приложений.

Создайте файл реестра со следующим содержимым, чтобы активировать компонент Imaging Viewer для форматов JPEG и PNG:

Windows Registry Editor Version 5.00


[HKEY_CLASSES_ROOT\Applications\imaging.dll\shell\open\command]


@="\"C:\\Windows\\System32\\rundll32.exe\" \"C:\\Windows\\System32\\shimgvw.dll\",ImageView_Fullscreen %1"



[HKEY_CLASSES_ROOT\.jpg\OpenWithList\imaging.dll]


@=""



[HKEY_CLASSES_ROOT\.png\OpenWithList\imaging.dll]


@=""

После применения этого скрипта в контекстном меню «Открыть с помощью» появится опция выбора классического просмотрщика. Чтобы сделать его стандартным, необходимо один раз выбрать его вручную через меню «Свойства» файла.

Этот метод особенно полезен на серверах с ограниченными ресурсами, где запуск тяжелого приложения Microsoft Photos может занимать избыточное количество оперативной памяти или процессорного времени.

Настройка через свойства файла

Если реестр изменен, но система продолжает предлагать выбор приложения, следует использовать диалоговое окно свойств. Нажмите правой кнопкой мыши на файл JPG, выберите «Свойства» и нажмите кнопку «Изменить» в поле «Приложение».

В открывшемся списке выберите «Найти другое приложение на этом компьютере». Перейдите в папку C:\Windows\System32 и выберите файл rundll32.exe (для классического метода) или найдите исполняемый файл Photos.

Важно установить галочку «Всегда использовать это приложение для открытия.jpg файлов». Это действие принудительно запишет ассоциацию в ветку реестра пользователя, переопределяя системные настройки по умолчанию.

Диагностика и устранение типовых проблем

Даже после выполнения всех настроек могут возникать ситуации, когда изображения перестают открываться или открываются в неправильном приложении. Чаще всего это связано с повреждением кэша ассоциаций файлов или конфликтом обновлений безопасности.

Первым шагом диагностики является проверка целостности системных файлов. Запустите командную строку от имени администратора и выполните команду sfc /scannow. Эта утилита проверит хэш-суммы системных библиотек, отвечающих за работу оболочки и графики.

Если проверка не выявила ошибок, но проблема сохраняется, стоит очистить кэш ассоциаций. Для этого можно воспользоваться утилитой assoc и ftype в командной строке, сбросив расширения к состоянию по умолчанию, а затем повторить процедуру настройки.

В некоторых случаях антивирусное ПО или политики AppLocker могут блокировать запуск UWP-приложений (таких как Photos). Необходимо проверить журналы событий Windows в разделе «Приложения и службы» → «Microsoft» → «Windows» → «AppxDeployment-Server».

  1. Ошибка «Приложение не запущено» — возникает при повреждении пакета приложения; требуется переустановка через PowerShell с флагом -Register.
  2. Открытие в браузере Edge — означает, что ассоциация сбита на уровень протокола HTTP или дефолтный handler браузера; требуется правка реестра для расширений.jpg/.png.
  3. Черный экран при открытии — часто вызвано отсутствием кодеков или конфликтом драйверов видеокарты в сеансе RDP; рекомендуется переключить renderer в настройках приложения.
  4. Невозможность выбрать приложение по умолчанию — блокировка групповыми политиками; проверьте наличие политик запрета изменения ассоциаций.
💡

Для быстрой диагностики используйте команду PowerShell Get-AppxPackage -Name Microsoft.Windows.Photos | Select InstallLocation, Version для проверки состояния пакета.

Работа в терминальном сеансе (RDP)

При работе через удаленный рабочий стол графические приложения могут работать медленнее из-за передачи bitmap-кадра. Приложение «Фотографии» может долго загружать превью изображений высокого разрешения.

Для оптимизации работы в RDP рекомендуется в настройках подключения (mstsc.exe) на вкладке «Взаимодействие» выбрать максимальную скорость соединения, чтобы отключить фоновые элементы рабочего стола и сглаживание шрифтов, что освободит канал для передачи изображения.

Также стоит проверить, не включен ли режим «Использовать аппаратное ускорение» в настройках самого приложения Photos, так как в виртуальных средах (Hyper-V, VMware) проброс GPU может быть не настроен, что вызывает зависания.

Сценарий Рекомендуемое решение Комментарий
Локальный сервер Microsoft Photos Полный функционал и поддержка
RDP / VDI Classic Viewer Меньше трафика, выше скорость
📌

Восстановление просмотрщика JPG в Windows Server 2019 требует установки UWP-компонента Photos или активации классического viewer через реестр.

Резюме и рекомендации по поддержке

Настройка стандартного просмотрщика изображений в Windows Server 2019 является необходимой операционной задачей для обеспечения комфортной работы администратора. Использование PowerShell для установки компонента Microsoft Photos является наиболее поддерживаемым Microsoft методом, гарантирующим совместимость с будущими обновлениями безопасности.

Для сред с высокими требованиями к производительности или в условиях ограниченного сетевого доступа целесообразно использовать классический просмотрщик через правку реестра. Это обеспечивает минимальное потребление ресурсов и независимость от внешних репозиториев.

Регулярный аудит установленных приложений и проверка ассоциаций файлов должны быть включены в стандартные процедуры обслуживания сервера. Автоматизация этого процесса через групповые политики позволяет поддерживать единую конфигурацию рабочей среды на всех узлах инфраструктуры.
