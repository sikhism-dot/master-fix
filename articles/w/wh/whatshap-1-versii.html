Руководство по использованию Whatshap версии 1.0 для фазирования гаплотипов

Работа с геномными данными часто упирается в необходимость разделения смешанных аллельных сигналов на отдельные родительские хромосомы, особенно при анализе данных секвенирования третьего поколения.

Исследователи, сталкивающиеся с короткими ридами Illumina или длинными прочтениями PacBio и Oxford Nanopore, обнаруживают, что стандартные пайплайны variant calling дают лишь список SNP без привязки к гаплотипам, что критично для изучения сложных геномных регионов.

Ситуация усугубляется, когда требуется интегрировать результаты в существующие биоинформатические цепочки, где несовместимость форматов файлов или версий зависимостей приводит к остановке анализа на этапе подготовки данных.

Архитектура и требования к окружению Whatshap 1.0

Инструмент whatshap версии 1.0 представляет собой консольную утилиту, написанную на Python и C++, предназначенную для фазирования генетических вариантов с использованием данных выравнивания.

Ключевой особенностью данной версии является поддержка алгоритмов, оптимизированных как для коротких, так и для длинных чтений, что требует специфической конфигурации вычислительного окружения для достижения максимальной производительности.

Для корректной работы программного обеспечения необходимо убедиться, что на сервере или рабочей станции установлены все системные библиотеки, включая libhts для работы с форматами сжатых геномных файлов.

Установка через менеджеры пакетов и исходный код

Наиболее надежным способом установки является использование менеджера пакетов Conda, который автоматически разрешает зависимости и создает изолированное окружение для работы с геномными данными.

Альтернативный метод предполагает компиляцию из исходных кодов репозитория, что может потребоваться при работе на специализированных кластерах с нестандартной архитектурой процессора или ограниченным доступом к внешним репозиториям.

Важно учитывать, что версия 1.0 требует Python версии не ниже 3.6, так как более старые интерпретаторы не поддерживают синтаксические конструкции, используемые в скриптах обработки аргументов командной строки.

  1. Создайте новое окружение командой conda create -n whatshap_env python=3.8 и активируйте его.
  2. Установите пакет из канала bioconda, выполнив команду conda install -c bioconda whatshap=1.0.
  3. Проверьте корректность установки, запустив команду whatshap --version в терминале.
  4. При необходимости обновите зависимости pysam и cython до актуальных стабильных версий.
📝

Если вы используете операционную систему macOS, убедитесь, что установлен Xcode Command Line Tools, так как компиляция некоторых модулей может требовать наличия нативных компиляторов C.

Подготовка входных данных: BAM и VCF файлы

Качество фазирования напрямую зависит от правильности подготовки входных файлов, основными из которых являются отсортированный BAM-файл с выравниваниями и файл вариантов в формате VCF или BCF.

Программа whatshap требует, чтобы BAM-файл был отсортирован по координатам и проиндексирован, иначе процесс чтения данных завершится ошибкой на этапе инициализации.

Файл вариантов должен содержать только те SNP и инделы, которые подлежат фазированию; inclusion лишних вариантов может существенно замедлить работу алгоритма без улучшения результата.

Требования к индексации и сортировке

Перед запуском основного процесса необходимо убедиться, что для BAM-файла создан индекс с расширением .bai, который позволяет алгоритму быстро обращаться к конкретным регионам генома.

Если входной VCF файл содержит множественные аллели в одной позиции, рекомендуется провести нормализацию данных, чтобы привести сложные варианты к каноническому представлению.

Для длинных чтений критически важно, чтобы в BAM-файле присутствовали теги, указывающие на качество выравнивания, так как алгоритм использует эту информацию для взвешивания доказательств.

samtools sort -o sorted.bam input.bam

samtools index sorted.bam


bcftools norm -m -any -f reference.fasta input.vcf > normalized.vcf

⚠️

Использование неиндексированных BAM-файлов приведет к значительному увеличению времени выполнения или аварийному завершению программы с кодом ошибки 1.

В процессе подготовки данных следует обратить внимание на наличие дубликатов PCR, которые могут исказить статистическую оценку фазы, если они не были помечены или удалены на предыдущих этапах.

Рекомендуется использовать инструменты вроде samtools markdup или picard MarkDuplicates для обработки артефактов амплификации перед передачей данных в whatshap.

  • Проверьте соответствие имен хромосом в BAM и VCF файлах (например, префикс chr).
  • Убедитесь, что кодировка файлов UTF-8 и отсутствуют скрытые символы в заголовках.
  • Отфильтруйте варианты с низким качеством (QUAL < 20) для повышения точности.
  • Проверьте, что покрытие генома равномерно и нет регионов с экстремально высоким глубиной.

Запуск фазирования и параметры алгоритма

Основной командой для запуска процесса является whatshap phase, которая принимает на вход BAM и VCF файлы и генерирует фазированный VCF на выходе.

Алгоритм использует динамическое программирование для минимизации количества переключений между гаплотипами, учитывая веса прочтений, перекрывающих гетерозиготные позиции.

Пользователь может гибко настраивать параметры, такие как максимальное расстояние между вариантами или минимальное количество прочтений, поддерживающих фазу.

Настройка параметров для разных типов данных

Для данных Illumina с короткими ридами стандартные параметры обычно работают хорошо, однако для длинных чтений PacBio HiFi или Nanopore требуется корректировка порога ошибки.

Параметр --ignore-read-groups позволяет игнорировать информацию о группах чтения, что может быть полезно при работе с данными, полученными из смешанных источников.

Использование флага --output обязательно для сохранения результатов, иначе программа попытается вывести огромный объем данных в стандартный поток вывода, что может переполнить буфер терминала.

📋 Базовый запуск фазирования

1Подготовьте reference.fasta, sorted.bam и variants.vcf
2Запустите whatshap phase --output phased.vcf sorted.bam variants.vcf
3Проверьте лог файл на наличие предупреждений
4Проанализируйте выходной VCF файл

В таблице приведены основные флаги командной строки, влияющие на поведение алгоритма в версии 1.0.

Параметр Описание Значение по умолчанию
--reference Путь к файлу референсного генома Не требуется (опционально)
--ped Файл родословной для семейного фазирования Отсутствует
--chromosome Имя хромосомы для обработки Все хромосомы

Семейное фазирование и использование PED файлов

Если доступны данные секвенирования родителей или других родственников, whatshap позволяет использовать эту информацию для повышения точности через механизм передачи аллелей.

Для этого необходимо подготовить файл в формате PED, описывающий структуру семьи, и передать его через аргумент --ped.

Такой подход позволяет разрешать фазу в регионах с низким покрытием, опираясь на наследование, что невозможно при использовании только данных одного индивидуума.

whatshap phase --ped family.ped --output family_phased.vcf sorted.bam input.vcf
💡

Для больших когорт рассмотрите возможность распараллеливания процесса по хромосомам, запуская отдельные задачи для каждой хромосомы независимо.

Анализ результатов и интерпретация output

Результатом работы программы является VCF файл, в котором в поле GT (Genotype) появляется разделитель | вместо /, указывающий на определенную фазу аллелей.

Кроме того, добавляются новые поля в заголовок и строки данных, такие как PS (Phase Set), которые группируют варианты, принадлежащие к одному блоку фазирования.

Важно понимать, что не все варианты могут быть фазированы; для некоторых позиций останется символ . или разделитель /, если доказательств недостаточно.

Статистика и метрики качества

Программа генерирует подробную статистику, включая количество фазированных блоков, среднюю длину блока и Switch Error Rate, если известен эталонный набор данных.

Анализ распределения длин блоков фазирования позволяет оценить качество входных данных: длинные блоки характерны для данных с длинными ридами или трио-анализа.

Для визуализации результатов можно использовать инструменты вроде IGV или специализированные библиотеки Python для работы с геномными данными.

Оценка результатов фазирования

👍 Преимущества
  • ✅ Высокая точность на длинных ридах
👎 Недостатки
  • ❌ Возможность работы с трио-данными

При интерпретации результатов следует учитывать, что блоки фазирования могут разрываться в регионах повторов или низкого качества секвенирования.

Статистическая достоверность фазы снижается при уменьшении глубины покрытия, поэтому для критически важных приложений рекомендуется пороговое значение покрытия не менее 20x.

Типичные ошибки и методы их устранения

В процессе эксплуатации whatshap версии 1.0 пользователи часто сталкиваются с рядом типовых проблем, связанных с форматом данных или нехваткой ресурсов.

Понимание причин этих ошибок позволяет быстро диагностировать проблему и продолжить работу без потери данных или необходимости перезапуска всего пайплайна.

Ниже приведен список наиболее распространенных сценариев сбоев и способы их корректного разрешения.

  1. Ошибка "IndexError: list index out of range" — возникает при несоответствии имен хромосом в BAM и VCF файлах; проверьте префиксы (chr1 против 1) и унифицируйте их.
  2. Ошибка "ValueError: invalid literal for int()" — указывает на повреждение VCF файла или наличие некорректных символов в поле качества; проведите валидацию файла через vcf-validator.
  3. Прерывание работы из-за нехватки памяти (OOM) — характерно для фазирования целых хромосом с высоким покрытием; используйте параметр --chromosome для разбивки задачи.
  4. Отсутствие фазирования (все генотипы 0/1) — означает, что алгоритм не нашел перекрывающихся ридов; проверьте, что BAM файл отсортирован и содержит данные для указанных в VCF позиций.
📝

При возникновении ошибок, не описанных в документации, попробуйте запустить программу с флагом --debug для получения расширенного лога выполнения.

Систематический подход к проверке входных данных перед запуском позволяет избежать большинства ошибок, связанных с форматированием.

Регулярное обновление зависимостей и использование стабильных версий библиотек htslib также снижает риск возникновения конфликтов на уровне системных вызовов.

Заключение

Использование whatshap версии 1.0 предоставляет исследователям мощный инструмент для точного фазирования гаплотипов на основе данных секвенирования нового поколения.

Грамотная подготовка входных файлов и понимание принципов работы алгоритма позволяют получать надежные результаты даже в сложных геномных регионах.

Внедрение этого инструмента в биоинформатические пайплайны значительно улучшает качество последующего анализа, включая поиск составных гетерозигот и реконструкцию гаплотипов.

📌

Whatshap 1.0 является стандартом де-факто для фазирования, обеспечивая баланс между точностью и вычислительной эффективностью при правильной настройке окружения.
